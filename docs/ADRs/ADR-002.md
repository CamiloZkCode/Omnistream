# ADR 002: Protocolo de Comunicación entre Monolito y Microservicio de Descuentos

## Estado
Aceptado

## Contexto

Una vez decidido extraer el MotorDescuentos a un microservicio independiente (ADR 001), es necesario definir cómo se comunicarán:
- **El monolito legado** (cliente) que necesita calcular descuentos
- **El nuevo microservicio** (servidor) que expone la lógica de descuentos

Requisitos técnicos identificados:
1. El monolito actual está escrito en Python (FastAPI para la API, CLI para consola)
2. El microservicio también se implementará en Python por consistencia del equipo
3. El contrato actual del MotorDescuentos recibe un objeto `usuario` y devuelve un `float` (precio final)
4. Se requiere latencia baja (< 150ms según escenario de disponibilidad)
5. El equipo tiene experiencia previa con REST
6. El presupuesto es limitado (no se justifica infraestructura compleja)

## Decisión

**Utilizaremos REST síncrono sobre HTTP/1.1 con JSON como formato de intercambio.**

### Especificación técnica

```http
POST /calcular-descuento
Content-Type: application/json

{
    "precio_base": 100.0,
    "codigo_vip": true,
    "antiguedad": 5
}

Respuesta:
{
    "precio_final": 67.5,
    "descuento_aplicado": 25
}
